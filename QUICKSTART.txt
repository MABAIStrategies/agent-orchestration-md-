WHAT THIS SKILL DOES
================================================================================

5 CORE PROTOCOLS:

1. SKILL SWEET SPOT PROTOCOL
   → Maintain 5-15 active skills to prevent context bloat
   
2. NEGATIVE CONSTRAINT ENGINEERING
   → Enforce opinionated design/code standards
   → No generic Bootstrap UI, pure black/white, or unfinished code
   
3. CHECKPOINT WORKFLOW
   → PLAN → MOCK → EXECUTE → VERIFY → DOCUMENT
   → Prevents misaligned expectations on complex tasks
   
4. SESSION HYGIENE
   → Rotate sessions after ~30 messages for clarity
   → Use .agent/notes.md for persistence between sessions
   
5. ARTIFACT STANDARDS
   → Professional naming: proposal-v2.1.pdf
   → Storage in artifacts/ or .agent/ folders
   → Metadata tagging for searchability

================================================================================
TROUBLESHOOTING
================================================================================

Q: "The constraints feel restrictive"
A: Use "NO CONSTRAINTS" to override. Constraints prevent generic output,
   not creativity. Customize them in SKILL.md section 2.

Q: "I keep forgetting to use these protocols"
A: Pin the quick reference card (SKILL.md section 9). Set calendar reminder
   for weekly skill audits. Ask Claude which protocol to use.

Q: "Checkpoint workflow seems slow"
A: Skip for simple tasks. Combine PLAN+MOCK in one prompt. Track: time in
   planning vs. rework — usually nets time savings.

Q: "My team isn't adopting this"
A: Start with Negative Constraints only (easiest ROI). Show metrics from
   week 1. Make it mandatory in checklists. Celebrate quick wins.

Full troubleshooting guide in IMPLEMENTATION_GUIDE.md section 5.
